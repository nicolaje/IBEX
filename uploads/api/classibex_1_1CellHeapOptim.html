Detailed Description

Cell Heap for Optimization.

The heap is organized so that the next box is always the one for which the evaluation of the criterion to minimized (a function f) gives an interval with the least lower bound. This criterion is stored as a variable (it is not re-calculated here).

ibex::CellHeapOptim::CellHeapOptim ( const int  y)

Build a cell heap for optimization.

Build a cell heap that stores (n+1)-dimensional boxes of the following form:
([x]_1,...[x]_n,[y])
where "y" is a specific variable that is assumed to be the criterion to be minimized. Typically, [y] is the image of a function f calculated on the box [x]=([x]_1,...[x]_n).

The heap is built so that:

  • pop() returns in logarithmic time the box ([x],[y]) with the minimal lower bound for [y]. among all the boxes in the list
  • push() is also in logarithmic time.
Parameters
y- the index of the variable "y" that contains the criterion (typically, f(x)) in each cell's box.
