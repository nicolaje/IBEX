Detailed Description

Global Optimizer.

This class is an implementation of the global optimization algorithm described in the AAAI'11 paper Inner Regions and Interval Linearizations for Global Optimization by Trombettoni et al.

Remarks
In all the comments of this class, "loup" means "lowest upper bound" of the criterion f and "uplo" means "uppermost lower bound" of the criterion.

Create an optimizer.

Parameters
sys- the extended system (see below)
bsc- bisector for extended boxes
ctc- contractor for the extended system

And optionally:

Parameters
prec- absolute precision for the boxes (bisection control)
goal_rel_prec- relative precision of the objective (the optimizer stops once reached). goal_abs_prec - absolute precision of the objective (the optimizer stops once reached).
sample_size- number of samples taken when looking for a "loup"

extended system (see copy constructor of #ibex::System) contains: (n+1) variables, x_1,...x_n,y. The index of y is #goal_var (==n). A (m+1)-valued function f:(x,y)->(y-f(x),g_0(x),...,g_m(x)) (m+1) constraints: y-f(x)=0, g_1(x)<=0, ..., g_m(x)<=0.

Warning
The optimizer relies on the contractor ctc to contract the domain of the goal variable. If this contractor never contracts this goal variable, the optimizer will never stop.

System ibex::Optimizer::sys

The normalized system.

Corresponds to the system (see constructor) with all inequalities under the form g_i(x)<=0.

ExtendedSystem ibex::Optimizer::ext_sys

The extended system.

Corresponds to the normalized system with the goal f(x) represented as a variable "y" and with the additional constraint y=f(x). The domain of y stores the interval [uplo,loup] where "uplo" is the uppermost lower bound on f(x) and "loup" the lowest upper bound.

The index of y is ext_sys.nb_var. See ibex::ExtendedSystem::goal_var.
