Standard installation (recommended)

Ibex can either be compiled with Gaol or Profil/Bias. The standard installation will automatically extract and build the Gaol library (and its dependencies) from the bundle. It will also make you install Soplex in a specific folder. Finally, it will install Ibex on your system, as a static 32-bits library (even if your platform is 64 bits).

But you can also {anchor anchor='options' text='customize'} the installation.

Linux and MacOS

The installation assumes your system meets the {anchor anchor='requirements' text='requirements'}.
  • Create a new directory Ibex
    ~$ mkdir Ibex
  • Download Soplex 1.7.0 and save the archive in Ibex
  • Compile Soplex
    ~$ cd Ibex
    ~/Ibex$ tar xvfz soplex-1.7.0.tgz
    ~/Ibex$ cd soplex-1.7.0
    ~/Ibex/soplex-1.7.0$ make ZLIB=false
    ~/Ibex/soplex-1.7.0$ cd ..
    [for MacOS or Linux 64 bits] Compile Soplex with the following option instead:
    ~/Ibex/soplex-1.7.0$ make ARCH=x86 ZLIB=false
  • Save the archive ibex-lib-2.0.tar.gz in Ibex
  • Compile Ibex
    ~/Ibex/$ tar xvfz ibex-lib-2.0.tar.gz
    ~/Ibex/$ cd ibex-2.0
    ~/Ibex/ibex-2.0/$ ./waf configure
    ~/Ibex/ibex-2.0/$ sudo ./waf install
Requirements
The following applications must be installed.
  • g++
  • gcc
  • flex
  • bison
  • python2.x
  • make
  • g++-multilib (only for 64 bits platform)
  • pkg-config (optionnal)
  • jdk (optionnal)

Windows

  • Install Mingw+Msys
    Select the following components to be installed:
    • C Compiler
    • C++ Compiler
    • MSYS Basic System
    • MinGW Developer ToolKit
  • Install Python2
  • Create a new directory Ibex (in the shell of Mingw)
    ~$ mkdir Ibex
  • Download Soplex 1.7.0 and save the archive in Ibex
  • Compile Soplex in Ibex (still in the shell of Mingw):
    ~$ cd Ibex
    ~/Ibex$ tar xvfz soplex-1.7.0.tgz
    ~/Ibex$ cd soplex-1.7.0
    ~/Ibex/soplex-1.7.0$ make ZLIB=false
    ~/Ibex/soplex-1.7.0$ cd ..
  • Save the archive ibex-lib-2.0.tar.gz in Ibex
  • Compile ibex (still in the shell of Mingw)
    ~/Ibex/$ export PATH="$PATH:/c/Python27"
    ~/Ibex/$ tar xvfz ibex-lib-2.0.tar.gz
    ~/Ibex/$ cd ibex-2.0
    ~/Ibex/ibex-2.0/$ ./waf configure --prefix=[path]
    where [path] is the place where you want Ibex to be installed. This path must be entered in Windows-style and with double backslash («\\») as separator, e.g., C:\\MyLibraires\\Ibex.
    Finally:
    ~/Ibex/ibex-2.0/$ ./waf install

Customized Installation

Configuration options

waf configure supports the following options:
  • --with-gaol=[path]
    Use the (already installed) version of Gaol at the given path.

    Warning: to get good performances it is highly recommended to have configured Gaol with the options --disable-preserve-rounding --enable-optimize.

    Note: The configuration will also automatically look for two other librairies required by Gaol: gdtoa and ultim (mathlib). It is assumed that these librairies are installed under standard system folders (like /usr/lib) or under [gaol-path]/lib. Otherwise, you have to set compiler flags (e.g., CPPFLAGS=-I/[ultim-path]/include and LDFLAGS=-L/[ultim-path]/lib).

  • --with-bias=[path]
    Compile Ibex with Profil/Bias (instead of Gaol). Note that the bundle only contains Gaol, not Profil/Bias. This library must already be installed on your machine at the given path.

    Note: To compile Profil/Bias on a MacOS 64 bits platform, unzip this file under [profil path]/config.

  • --with-soplex=[path]
    Look for Soplex at the given path instead of the parent directory.

  • --enable-shared
    Compile Ibex as a dynamic library. See {anchor anchor='dynamic-install' text='the guidelines'}.

  • --with-jni
    Compile Ibex with the Java interface used by Choco.
    Ibex itself will be compiled as a dynamic library (as if configured with --enable-sharerd). See {anchor anchor='dynamic-install' text='the guidelines'}.

  • ---with-java-package=[package name]
    To be used with --with-jni only. Set the Java package name of the Ibex class. By default, the package is ibex.

  • --prefix=[path]
    Set the folder where Ibex must be installed (by default, /usr/local). You can use this option to install Ibex in a local folder.

Installation as a dynamic library (--enable-shared or --with-jni)

You can install Ibex as a dynamic library, but with Gaol only. Of course, it is technically possible to compile Ibex+Profil as a dynamic (and 64-bits) library, but you have to do it by yourself (no script supplied).

There are only a few differences with the standard installation:

  • Linux and MacOS
  • Compile Soplex as follows:
    ~/Ibex/ibex-2.0/soplex-1.7.0$ make ARCH=x86 SHARED=true ZLIB=false
  • Windows
    Compile Soplex as follows:
    ~/Ibex/ibex-2.0/soplex-1.7.0$ make ZLIB=false SHARED=true SHAREDLIBEXT=dll

Then, you can run waf configure with either --enable-shared or --with-jni.

Installation of the Java interface (--with-jni)

Important notice:

  • This interface is under development
  • This option is currently not supported under 64 bits platform unless you have installed a 32 bits JVM. This is because Gaol currently supports only 32 bits platform.
  • The Ibex-Choco interface is not supported yet by the current release of Choco.

The only additional requirement is that the environment variable JAVA_HOME must be set. Typical paths are /Library/Java/Home (MacOS) or /usr/lib/jvm/java-7-openjdk-i38 (Linux). Example:

~/Ibex/$ export JAVA_HOME=/Library/Java/Home

This installation will generate, in addition to the Ibex library:

  • the libibex-java library that contains the glue code between C++ and Java
  • the [java package].jar file where [java package] is the name given with the --with-package option. This file is put into the [prefix]/share/java where [prefix] is /usr/local by default or whatever path specified via --prefix.

Compiling and running examples

Using the Makefile (recommended)

If you have installed Ibex following the {anchor anchor='stand-install' text='standard installation'}, there is a simple makefile you can use to compile examples as well as your own programs. This makefile however assumes pkg-config is installed on your system (which is done by default on many Linux distribution).

If you have installed Ibex in the local folder, or if you don't want pkg-config to be installed, you may consider the {anchor anchor='basic-make' text='basic command line'} instead.

Note: it may be necessary to set the PKG_CONFIG_PATH to [prefix]/share/pkgconfig where [prefix] is /usr/local by default or whatever path specified via --prefix:

~/Ibex/ibex-2.0/$ export PKG_CONFIG_PATH=/usr/local/share/pkgconfig/

Note: if some libraries are linked dynamically (like libultim), it may be necessary to set the LD_LIBRARY_PATH accordingly.

Here is an example:

~/Ibex/ibex-2.0/$ cd examples
~/Ibex/ibex-2.0/$ make defaultsolver
~/Ibex/ibex-2.0/$ ./defaultsolver ../benchs/cyclohexan3D.bch 1e-05 10

The default solver solves the systems of equations in argument (cyclohexan3D) with a precision less than 1e-05 and within a time limit of 10 seconds.

To compile your own program, just copy-paste the makefile of Ibex/examples.

Note: this makefile uses the extended syntax of GNU make.

Basic command line

You can alternatively try the following command line directly, but the success is less guaranteed. The basic line is:

g++ -m32 -I[prefix]/include/ibex -I[soplex path]/src -L[soplex path]/lib -libex -lsoplex -lz
     -o [filename] [filename].cpp
where [prefix] is /usr/local by default or whatever path specified via --prefix.

  • If your platform is 64 bits
    Add the -m32 option
  • If Ibex is installed in a local folder
    • Add -L[ibex path]/__build__/src
  • If you have compiled Ibex with Profil/Bias
    • Add -I[profil path]/include
    • Add -L[profil path]/lib
    • At the end of the line, add
      -lProfil -lBias -llr -lsoplex -lz

Java examples

under development

If you have installed Ibex with the --with-jni option, you can run a java test example.

  • Linux
    ~$ export LD_LIBRARY_PATH=[prefix]/lib
    where [prefix] is /usr/local by default or whatever path specified via --prefix.
    Then:
    ~$ cd Ibex/ibex-2.0/__build__/src/java
    ~/Ibex/ibex-2.0/__build__/src/java$ $JAVA_HOME/bin/java -Djava.library.path=$LD_LIBRARY_PATH Test
  • MacOS
    ~$ export DYLD_LIBRARY_PATH=[prefix]/lib
    where [prefix] is /usr/local by default or whatever path specified via --prefix.
    Then:
    ~$ cd Ibex/ibex-2.0/__build__/src/java
    ~/Ibex/ibex-2.0/__build__/src/java$ $JAVA_HOME/bin/java -Djava.library.path=$DYLD_LIBRARY_PATH Test

Notice: $JAVA_HOME ensures the JVM is compatible with the version of Java you have compiled Ibex with.

Uninstall

Simply type in the path of IBEX (under the shell of Msys for Windows)

~/Ibex/ibex-2.0$ ./waf uninstall
